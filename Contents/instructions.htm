Steampunk CPU/GPU Thermometer Yahoo widget

Installation Instructions

This Widget will install and run on your Windows XP/Win7 system and the thermometers will return valid temperatures. Although this version of the widget will install on Mac OS/X it will not return any valid temperature information. A later version of the widget will work on Mac OS/X operating system.

This widget uses Speedfan to obtain temperature information. Why is Speedfan needed? The answer is that the Windows operating system is meant to provide an interface to the motherboard and other components to provide system temperature information. Unfortunately, Windows does not yet do its job as there seems to be no stable hardware specification for hardware manufacturers to provide  temperature sensors. All the hardware manufacturers do their own thing. The end result is that it is impossible for a widget to get the temperatures from all the myriad different types of PC components in the world, so we have to resort to third party applications that can do it pretty well. The best of the bunch is a good piece of software called Speedfan. Speedfan is pretty good at extracting temperatures for almost all PC hardware so it is a fundamental requirement for Speedfan to be installed on your Windows PC for this widget to function.

Get Speedfan here: http://www.almico.com/sfdownload.php

You will  need Yahoo widgets for this widget to run. Get Yahoo widgets here.  This widget is written in Javascript and XML which is used by the Yahoo widget runtime engine. This widget is capable of running under the Yahoo widget engine only.

Startup

On Windows XP Speedfan will be started and minimised straight away. On Win 7/Vista you may have to minimise it manually the first time it runs. This is due to the bug where NT6 restricts access to Speedfan's installation folder. When Speedfan is started by the widget there will be a wait for 15-20 seconds whilst Speedfan interrogates all the sensors. When this is done it will start to show the temperatures.

Instructions for use

The temperatures of your CPU/GPU are indicated by the level of the red alcohol (or mercury) on each thermometer. The maximum temperature reached by the thermometer is shown by the metal index that floats above the mercury level. Click and slide to reposition it.

 The maximum temperature reached by the thermometer is shown by the metal index that floats above the mercury level. Click and slide to reposition it.
To set the alarms you simply slide the warm and hot slider settings to adjust the alarms to appropriate temperatures for your particular laptop. Click on each slider and slide to adjust. The user defined slider positions will be saved in future versions.When the temperature exceeds the setting for the warm slider the amber alarm will flash. When the temperature exceeds the setting for the hot slider the red alarm will flash and the bell alarm will sound.
 
To silence the alarm, simply click on the bell. The clapper will move away from the bell indicating the alarm setting is off. This can also be set at the back end in Widget Preferences.
The sampling interval is 5-6 seconds, move the oval slider on the time selector to reduce the sampling interval and increase the frequency of updates. This can also be set at the back end in Widget Preferences.
Cranking the handle on the bottom right will cause the printer to unfold. This will display a chart of the temperatures over the time the widget has been running. This chart is not yet fully working. Crank the handle again and the printer will fold up again.This can also be set at the back end in Widget Preferences.
  The printer scribes move up and down corresponding to the current temperature. The CPU scribe is highlighted with a capital C. The GPU with a capital G. The scribes are not yet fully working in this beta version.
 
Clicking on the H toggle will display the drop down help canvas.
 
Clicking on the S toggle will take you straight to the Speedfan site to download Speedfan.
 
The widget has its own set of sounds that make the thermometer widget unique and humourous. These sounds are not always required when you are listening to music or any other program with its own sounds. You can mute the thermometer by turning off the sounds from the back end.

The code behind the steampunk thermometer  allows the yahoo widget to be resized. It can now be made significantly smaller than the standard large size and it looks rather sweet in small form. Not everything will work after a resize though! The widget can only be resized at the back end in Widget Preferences.

 

 

 

 

 

 

Sensor Selecting Menu

If you perform a right click on the widget it will pop up a drop down menu. The menu is context-sensitive so if you right click when on the thermometer body more options will appear. These are the alternative sensors that can be used by the thermometer. By default one of the sensors will be selected by the program logic that determines the best one to use.

Typically, a number of sensors are shown, the image to the left shows some typical sensors:

Sensor 1 GPU sensor (assuming Nvidia)
Sensor 2-3 Hard discs (assuming 2 hard discs)
Sensor 4 Temp 1 - unidentified functioning ACPI sensor (probably motherboard)
Sensor 5 CPU core 1
Sensor 6 CPU core 2 (could be as many as four)
Sensor 7 GPU sensor read through Nvidia control panel (should be the same as sensor 1)
Sensor 8 unidentified non-functioning sensor

The menu allows you to select any sensor manually by clicking and selecting the sensor of choice.

Sensor Title Menu 


 

As the widget cannot yet determine the name of the sensor from the information provided by speedfan you'll need to change the thermometer title yourself. This is the title at the top of the thermometer. Simply right-click on the title and you will be presented with a drop down that allows you to select from a list the appropriate sensor titles. Note: this just changes the name, nothing else, it does not select a different sensor. Use the Sensor Selecting Menu above to do that.

 

 

 

 

 

 


Run-time Problems

On Windows XP - the widget can determine the sensor names and will return the correct temperature for CPU and GPU. However, on NT6 systems (win 7 and Vista) the temperatures returned will initially be that of the GPU and one other 'unknown' sensor. The widget will do its best to determine the correct one. However, we cannot currently guarantee it is the CPU temperature as NT6 does not allow you to read the Speedfan installation folder where the information is stored. Also, due to the nature of the Speedfan API, Speedfan does not return the names of sensors but only the temperature values. This is really annoying as it means we cannot be sure which temperature we are returning. At the moment the widget will do its best to guess and in any case the widget gives you the functionality to decide which sensor to use. In the long term the developer of Speedfan will almost certainly be making changes to improve his program and enable the above functionality. We will just have to wait a while longer until he does this. Sorry!
